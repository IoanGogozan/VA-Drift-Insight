# Portfolio Screenshots

Use these screenshots in a portfolio PDF, GitHub README, job application attachment or interview presentation.

The goal is to show a complete workflow:

```text
Oversikt -> Vannsoner -> Lekkasjekontroll -> Private saker -> Feltoppgaver -> Rapport -> Datagrunnlag
```

## Portfolio PDF Structure

Recommended 4-6 page structure:

1. Project summary and purpose
2. Main workflow screenshots
3. Backend and database architecture
4. Scoring and data quality
5. Report output
6. What the project demonstrates about the developer

## Quality Checklist

- Screenshots should be taken with seeded demo data loaded.
- Browser zoom should be 100%.
- Avoid screenshots with browser console errors.
- Use the same viewport size for all desktop screenshots.
- Do not show real infrastructure data or private information.
- Prefer Norwegian UI screenshots because the target market is Norway.
